## A. РАСХОЖДЕНИЯ (что есть в инструкции, но неверно/устарело)

### A1. §1 Авторизация — УСТАРЕЛО (outdated, подтверждено)
- **Claim:** «нажмите «Настроить API ключи» … введите в окне настроек»
- **Реальность:** API-ключи вводятся на **вкладке «Подключение»** прямо на экране входа (login_view.py:196-267). Кнопки «Настроить API ключи» больше нет.
- **Fix:** «При первом запуске откройте вкладку «Подключение» (рядом с «Вход»). Получите api_id/api_hash на my.telegram.org → API development tools. Введите в поля и нажмите «Сохранить».»

### A2. §3 Фильтр по автору — НЕВЕРНО (wrong, проверено лично)
- **Claim:** «экспортировать только сообщения от конкретного пользователя (по username или ID)»
- **Реальность:** В export_modal.py НЕТ UI для фильтра автора. get_export_options (383-392) не отдаёт author. AuthorFilter существует в коде (export_task.py:32-46) и применяется в orchestrator.py:205-210, но задать его из UI нельзя.
- **Решение нужно:** удалить пункт ИЛИ реализовать поле в ExportModal.

### A3. §6 profiles.json — НЕ УПОМЯНУТ (outdated, подтверждено перепроверкой)
- **Реальность:** ~/.tg_exporter/ теперь содержит profiles.json (profiles.py:30) — список аккаунтов.
- **Fix:** добавить «profiles.json (список аккаунтов)» в перечень файлов.

## B. НЕТОЧНОСТИ (imprecise — верно по сути, мелкие уточнения)

- **§2 Период:** на главном экране подпись «Период (папка)» — применяется только к экспорту папки; для одного чата период в окне экспорта (chats_page.py:99). Формат в подсказке `YYYY-MM-DD`, а не `ГГГГ-ММ-ДД` (date_range_row.py:14).
- **§4 Транскрипция:** чекбокс «Транскрипция (папка)» стоит ПОСЛЕ выбора режима, не «рядом с кнопкой»; есть иконка «?» с подсказкой (chats_page.py:133-161).
- **§5 Модель Whisper:** UI показывает 5 моделей (Tiny/Base/Small/Medium/Large), Base='баланс' не 'быстро'; config поддерживает large-v2/v3 (settings_page.py:43-49, config.py:20).
- **§5 Язык:** опция отображается как «Авто (несколько языков)» (settings_page.py:32-41).
- **§7 Отмена:** [ПОДТВЕРЖДЁН И ИСПРАВЛЕН] При отмене JSON-экспорта result.json оставался
  невалидным — close() (дописывает `]}`) не вызывался: в _do_run не было try/except вокруг
  цикла, CancelledError летел в run() мимо close(). Markdown НЕ затронут (буферизует в памяти,
  файлы пишет только в finalize, при отмене на диске ничего нет). Фикс: try/except CancelledError
  в _do_run закрывает экспортёры перед пробросом. Тест: tests/test_orchestrator_cancel.py.
